What this dashboard is

A live read of today, not a forecast

The Daily command centre sits on top of the same data /daily-plan writes to 01_Inbox/Daily_Plans/{date}.md. The markdown file is the audit log. This artifact is the interactive surface: a single pane for what matters today, what is drifting in Salesforce, what is on the calendar, what is in the punch list, and what is sitting in the inbox.

Drift detection logic

How the chip is calculated

Each calendar event tagged to a Salesforce opportunity has a baseline snapshot of StageName, CloseDate, Net_ARR__c, and SE_Risk_Level__c captured when /daily-plan ran. At view time, the artifact runs a single SOQL query to fetch the current values for all tagged opportunities, then compares field by field:

  • Aligned: all four fields match the baseline. Green chip.
  • Drifted: any field disagrees. Pank chip. Hover the row to see the specific diffs.
  • Unavailable: Salesforce is not reachable or the opportunity ID is no longer queryable. Grey outlined chip.

Drift detection is descriptive, not prescriptive. A drifted chip means the workstation memory is stale, not that the deal has changed for the worse.

Source of truth
  • Salesforce for stage, close date, net ARR, risk level. Always.
  • 02_Tasks/Tasks.md for the punch list. Edits in the artifact propose changes, they do not commit.
  • System/user-profile.yaml for timezone, role, fiscal calendar. Edit there if these are wrong.
  • System/pillars.yaml for pillar names. Pillar pills on the Tasks tab read from this list.
What this is not
  • Not a replacement for the markdown brief. 01_Inbox/Daily_Plans/{date}.md is still written and still read by /daily-review, /week-plan, and /week-review.
  • Not a pipeline review. That surface is owned by /avp-pipeline-review. Drift chips here are a tripwire, not a full read.
  • Not a cross-day rollup. This is today only. Weekly views live in /week-plan and /week-review.
  • Not a write-through. Every persistent change goes back through chat for review before disk.
